### 安全实践说明

1. **参数化查询**
   - 所有原生 SQL 查询都使用了 `%s` 占位符
   - 参数通过 `cursor.execute(sql, params)` 方式传递
   - 数据库驱动会自动转义特殊字符

2. **Django ORM 防护**
   - 使用 `filter()`, `get()` 等 ORM 方法
   - Django ORM 自动处理参数转义
   - 避免字符串拼接 SQL

3. **输入验证**
   - 序列化器对输入数据进行校验
   - 手机号、序列号等字段有格式验证
   - 字符串长度限制
