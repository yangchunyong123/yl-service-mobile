# 📱 售后现场移动终端系统开发需求提示词

## 🎯 核心目标
基于公司光伏组件售后服务量增加的现状，开发一款适用于电站现场的移动终端系统（APP或Web），旨在替代纸质/零散电子记录，实现数据的实时上传、集中管理与全程追溯，从而提升现场服务效率与数据准确性。

## 📄 项目背景
- **现状痛点**：现场维修、换货处理依赖纸质或零散电子记录，数据整合困难，后续追溯不便。
- **部门诉求**：质量管理部申请信息管理部开发该系统，需配合提供测试场景与反馈。

---

## 🛠️ 核心功能需求

### 1. 现场扫码查询功能
系统需支持通过扫描条码查询并下载以下信息：

| 功能模块 | 详细要求 |
| :--- | :--- |
| **EL图纸查询** | 支持扫码后查看并下载组件的 EL 检测图像。 |
| **电子流转单** | 支持下载流转单（格式不限），用于现场核对与备案。 |
| **组件明细报表** | 支持连续扫码多个条码，查询并导出包含以下信息的报表：<br>1. **电性能参数**（含组件功率）<br>2. **材料信息**（电池片、胶膜、边框、接线盒的厂家名称及规格型号）<br>3. **业务信息**（合同、客户信息，视系统情况决定是否显示） |

### 2. 客诉信息录入与上传
开发《组件维修信息表》电子表单，支持现场扫码或手动录入。数据需实时同步至后台，并支持后续扫码查询维修记录。

**录入字段要求：**
- **基础信息**：客诉处理人、序列号、项目名称（公司+地点+容量+类型）、项目地点（市/县）。
- **判定信息**：是否质保（是/否）、问题种类（如雷击、击穿、碎裂、功率异常等）、方阵记录（逆变器信息）。
- **处理信息**：处理类型（维修/换货/提供备件/赔款）、换货序列号。
- **维修详情**：维修情况（需区分正极、中间、负极的处理状态，如“更换”、“无”）、维修人。
